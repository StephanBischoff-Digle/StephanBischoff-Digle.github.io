magazino/Notifier Bot

Magazino

The Magazino GmbH is a robotics startup based in Munich, Germany. The products of the company are autonomous logistics robots that get deployed in the warehouses of the customer to reduce the workload for the employees during times with a high amount of orders (christmas for example) or during times when the employees are on vacation or a high rate of sickness related drop outs is to be expected. The robots, once configured, can be dropped into the warehouse. New robots can use the configurations of the already deployed machines, such that the number of robots in the warehouse could be, in theory, very dynamic.

The Project

Description

The Magazino robots (TORU, SOTO) had to communicate errors somehow; especially during performance evaluations. For this purpose the robot had LEDs in front and in the back, such that errors could be signaled visually from afar. Additionally, operators connected to the robot via ssh were able to collect more detailed information over a tmux session dedicated to logging the robots state.

This kind of informationis useful as soon as it is generated and to more than just the person connected to the machine (which might be zero). The Notifier Bot checked the state of the robot in configured intervals and send them via specified channels (a chat channel or email, for example).

Contribution

The original Notifier Bot was created in a hurry as a convoluted and undocumented script that requested date from the robots database and send it to a Mattermost channel. My task was to extract long term value from this implementation by refactoring the script into a more elaborate architecture which allowed the bot to be extended.

I rewrote it using a message queue architecture. There is a base class for data sources and a base class for data sinks. The sources push data into the push data into the queue, which sanitizes the data and pushes it to the sink if necessary.

Technologies

Python 2 – Django – Python Mattermost Driver – git – GitLab – JIRA
